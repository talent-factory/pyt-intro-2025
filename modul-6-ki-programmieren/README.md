# Modul 6: KI-gestütztes Programmieren

**Dauer:** 1 Tag (4 Lektionen à 50 Minuten)
**Voraussetzung:** Module 1-5 abgeschlossen, GitHub-Account

## 🎯 Lernziele

Nach diesem Modul können Sie:

1. ✅ Die KI-Landschaft für Programmierer verstehen (Chat, IDE, CLI)
2. ✅ Claude Code installieren und konfigurieren
3. ✅ Effektive Prompts für Programmieraufgaben formulieren
4. ✅ Code mit KI-Unterstützung erklären, debuggen und generieren lassen
5. ✅ CLAUDE.md als Projektkontext verstehen und einsetzen
6. ✅ Grenzen und Verantwortung beim KI-gestützten Programmieren kennen
7. ✅ Fortgeschrittene Möglichkeiten (Plugins, Skills) als Ausblick einordnen

## 📖 Themen im Detail

### KI-Landschaft für Programmierer
- Chat-basierte KI-Tools (Claude.ai, ChatGPT)
- IDE-Integrationen (GitHub Copilot, Cursor)
- CLI-Tools (Claude Code)
- Stärken und Schwächen der verschiedenen Ansätze

### Prompting-Grundlagen
- Was ist ein Prompt?
- Gute vs. schlechte Prompts
- Spezifische Anweisungen formulieren
- Iteratives Prompting

### Claude Code in der Praxis
- Installation und Konfiguration
- Code erklären lassen
- Debugging mit KI-Unterstützung
- Code generieren und bewerten

### Projektkontext und CLAUDE.md
- Was ist CLAUDE.md?
- Einfluss auf KI-Output
- Eigene CLAUDE.md schreiben
- Best Practices für Projektkontext

### Grenzen und Verantwortung
- Wo KI versagt (Halluzinationen)
- Wann man KI-Output prüfen muss
- Ethische Aspekte
- KI als Werkzeug, nicht als Ersatz

## 📚 Modulstruktur

### [00-vorbereitung](./00-vorbereitung/) - Vorbereitung (2-3 Std)

**Vor dem Präsenztag:**
- Leseauftrag: KI-gestütztes Programmieren verstehen
- Erste Experimente mit KI-Chat-Tools
- Quiz zur Selbstkontrolle (10 Fragen)
- GitHub-Account bereithalten

### [01-praxis](./01-praxis/) - Präsenzunterricht (4 × 50 Min)

**Am Kurstag:**
- **Lektion 1:** KI als Programmierpartner (50 Min)
- **Lektion 2:** Claude Code: Erste Schritte (50 Min)
- **Lektion 3:** Effektiv mit Claude Code arbeiten (50 Min)
- **Lektion 4:** Showcase: Was noch möglich ist (50 Min)

### [02-uebungen](./02-uebungen/) - Übungen während der Präsenz

**Praktische Übungen:**
1. Prompt-Werkstatt
2. Code erklären lassen
3. Bug-Jagd mit KI
4. KI-gestützt programmieren
5. Lösungsvergleich
6. CLAUDE.md schreiben
7. Refactoring mit KI
8. KI-Grenzen testen

### [03-nachbearbeitung](./03-nachbearbeitung/) - Hausaufgaben (4-6 Std)

**Nach dem Präsenztag:**
- Aufgabe 1: KI-Tagebuch
- Aufgabe 2: Prompt-Sammlung
- Aufgabe 3: Tool-Vergleich
- Persönliche Reflexion

### [04-materialien](./04-materialien/) - Referenzmaterialien

**Nachschlagewerke:**
- Cheatsheet Claude Code
- Prompt-Guide
- Tool-Übersicht
- Referenz Plugins

### [05-beispiele](./05-beispiele/) - Beispielprogramme

**10 vollständige Python-Programme:**
- prompt_beispiele.py - Gute vs. schlechte Prompts als Code-Kommentare
- code_mit_bugs.py - Funktionen mit versteckten Fehlern
- ki_generiert_beispiel.py - KI-generierte Lösung mit Annotationen
- vorher_nachher.py - Eigene vs. KI-Lösung im Vergleich
- claude_md_beispiel.py - Einfluss von CLAUDE.md auf KI-Output
- passwort_generator.py - Musterlösung KI-gestützte Entwicklung
- refactoring_beispiel.py - Code vor/nach KI-Refactoring
- ki_grenzen.py - Wo KI versagt
- iteratives_prompting.py - Schrittweiser Verfeinerungsprozess
- debugging_mit_ki.py - Bugs finden mit KI-Unterstützung

## 💡 Tipps für den Erfolg

- **Vorbereitung ist wichtig:** Nehmen Sie sich Zeit für den Leseauftrag
- **Experimentieren Sie:** Probieren Sie verschiedene Prompts aus
- **Kritisch bleiben:** KI-Antworten immer hinterfragen
- **Eigene Erfahrung:** Selbst ausprobieren statt nur zuschauen
- **Verantwortung:** Sie bleiben für Ihren Code verantwortlich
