# Aufgabe 1: KI-Tagebuch

**Zeitaufwand:** 2 Stunden
**Schwierigkeit:** ⭐⭐ Mittel

## 🎯 Ziel

Entwickeln Sie ein kleines Programm komplett mit KI-Unterstützung und dokumentieren Sie jeden Schritt des Prozesses.

## 📝 Anforderungen

### Basis-Features (Pflicht)

1. **Aufgabe definieren** — Wählen Sie ein kleines Projekt:
   - Einkaufszettel-Manager
   - Quiz-Generator
   - Notiz-Verwaltung
   - Oder eine eigene Idee
2. **Mit Claude Code Schritt für Schritt entwickeln**
   - Starten Sie mit einer klaren Aufgabenbeschreibung
   - Lassen Sie sich den Code schrittweise generieren
   - Stellen Sie Rückfragen, wenn etwas unklar ist
3. **Jeden Dialog-Schritt dokumentieren:**
   - Was war Ihr Prompt?
   - Was hat die KI geantwortet?
   - Wie bewerten Sie die Antwort? (gut/mittel/schlecht)
4. **Endresultat testen und lauffähig abgeben**

### Erweiterte Features (Optional)

5. **Vergleich:** Hätten Sie es ohne KI schneller geschafft?
6. **Code-Review:** Eigene Verbesserungen am KI-generierten Code

## 💡 Beispiel-Ablauf

```
=== KI-TAGEBUCH ===

Projekt: Einkaufszettel-Manager

--- Schritt 1 ---
Prompt: "Erstelle mir eine einfache Einkaufsliste in Python,
         bei der ich Artikel hinzufügen und anzeigen kann."

KI-Antwort: [Code mit add/show Funktionen]

Bewertung: ⭐⭐⭐⭐ Gut
Kommentar: Code funktioniert, aber keine Fehlerbehandlung.

--- Schritt 2 ---
Prompt: "Füge bitte eine Fehlerbehandlung hinzu und die
         Möglichkeit, Artikel zu entfernen."

KI-Antwort: [Erweiterter Code]

Bewertung: ⭐⭐⭐⭐⭐ Sehr gut
Kommentar: Genau was ich wollte.

--- Schritt 3 ---
...
```

## 📋 Dokumentations-Vorlage

Verwenden Sie diese Struktur für Ihr Tagebuch:

```markdown
# KI-Tagebuch: [Projektname]

## Projektbeschreibung
[Was soll das Programm können?]

## Schritt 1: [Titel]
- **Prompt:** [Ihr Prompt]
- **Zusammenfassung der Antwort:** [Was hat die KI vorgeschlagen?]
- **Bewertung:** ⭐ bis ⭐⭐⭐⭐⭐
- **Kommentar:** [Was war gut/schlecht?]
- **Änderungen:** [Was haben Sie selbst angepasst?]

## Schritt 2: [Titel]
...

## Fazit
- Gesamtdauer: ___ Minuten
- Anzahl Prompts: ___
- Zufriedenheit mit dem Ergebnis: ⭐ bis ⭐⭐⭐⭐⭐
- Wichtigste Erkenntnis: ___
```

## ✅ Checkliste

### Basis
- [ ] Projekt gewählt und beschrieben
- [ ] Mindestens 5 Dialog-Schritte dokumentiert
- [ ] Jeden Prompt notiert
- [ ] Antworten bewertet
- [ ] Fertiges Programm lauffähig

### Erweitert
- [ ] Vergleich mit/ohne KI
- [ ] Eigene Code-Verbesserungen
- [ ] Reflexion über den Prozess
- [ ] Tipps für zukünftige KI-Nutzung
- [ ] Schöne Formatierung der Dokumentation

## 🎯 Lernziele

- KI als Entwicklungswerkzeug einsetzen
- Prompts iterativ verbessern
- KI-generierten Code kritisch bewerten
- Den Entwicklungsprozess dokumentieren
- Eigene Anpassungen am Code vornehmen

## 💻 Hilfe

### Projekt starten

```
claude
> Ich möchte ein Programm erstellen, das [Beschreibung].
> Bitte erstelle den Code Schritt für Schritt.
```

### Gute Prompts für den Prozess

```
"Erkläre mir, was dieser Code macht."
"Füge bitte Kommentare hinzu."
"Wie kann ich Fehlerbehandlung ergänzen?"
"Teste den Code mit diesen Eingaben: ..."
```

### Tagebuch speichern

Speichern Sie Ihr Tagebuch als `ki-tagebuch.md` und den fertigen Code als `.py`-Datei.

## 📦 Abgabe

**Dateien:**
- `ki-tagebuch.md` — Ihr dokumentierter Prozess
- `[projektname].py` — Ihr fertiges Programm

**Testen Sie:**
- Läuft das Programm fehlerfrei?
- Sind alle Schritte dokumentiert?
- Ist die Dokumentation nachvollziehbar?
