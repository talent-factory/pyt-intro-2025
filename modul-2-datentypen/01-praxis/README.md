# Praxis: Modul 2

**Dauer:** 4 Lektionen à 50 Minuten

## 🎯 Lernziele

Nach den Praxis-Lektionen können Sie:

- Strings erstellen, formatieren und manipulieren
- Boolsche Werte und Vergleichsoperatoren anwenden
- Typkonvertierung durchführen
- Interaktive Programme mit `input()` erstellen
- Erweiterte String-Methoden und Validierung anwenden

## 📋 Übersicht

Jede Lektion folgt dem gleichen Aufbau:

- **Theorie** (15 Min.) - Konzepte verstehen
- **Live-Coding** (20 Min.) - Dozent zeigt Best Practices
- **Übung** (15 Min.) - Studierende üben selbst

## 🎓 Didaktisches Konzept

Jede Lektion folgt dem gleichen Muster:

### 1. Theorie (15 Min)

- Konzept erklären
- Syntax zeigen
- Anwendungsfälle besprechen
- Häufige Fehler aufzeigen

### 2. Live-Coding (20 Min)

- Dozent programmiert live
- Studierende folgen mit
- Fragen werden direkt beantwortet
- Code wird gemeinsam getestet

### 3. Übung (15 Min)

- Studierende lösen Aufgabe selbstständig
- Dozent unterstützt bei Problemen
- Lösung wird gemeinsam besprochen
- Variationen werden diskutiert

## 💡 Tipps für Dozierende

### Zeitmanagement

- **Pünktlich starten:** Jede Lektion beginnt zur vollen Stunde
- **Pausen einplanen:** 10 Min nach 2 Lektionen
- **Puffer einbauen:** Nicht zu viel Stoff in eine Lektion packen
- **Flexibel bleiben:** Bei Bedarf Tempo anpassen

### Interaktion

- **Fragen stellen:** Studierende aktiv einbeziehen
- **Live-Coding:** Fehler machen und korrigieren (lehrreich!)
- **Pair Programming:** Studierende arbeiten zu zweit
- **Code Review:** Lösungen gemeinsam besprechen

### Fehlerkultur

- **Fehler sind normal:** Betonen Sie, dass Fehler zum Lernen gehören
- **Debugging zeigen:** Wie findet man Fehler?
- **Fehlermeldungen lesen:** Lernen Sie, Fehlermeldungen zu verstehen
- **Hilfe zur Selbsthilfe:** Zeigen Sie, wie man Lösungen findet

## 🆘 Häufige Probleme

### Problem 1: Studierende kommen nicht mit

**Lösung:**
- Tempo reduzieren
- Mehr Erklärungen
- Zusätzliche Beispiele
- Pair Programming

### Problem 2: Studierende sind zu schnell

**Lösung:**
- Zusatzaufgaben bereitstellen
- Fortgeschrittene helfen Anfängern
- Bonusaufgaben anbieten

### Problem 3: Technische Probleme

**Lösung:**
- Codespaces als Backup
- Studierende helfen sich gegenseitig
- Notfalls Code auf Papier schreiben
